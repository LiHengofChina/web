//================ 外部js文件的优势？

	如果两个页面都使用同一个文件，
	那么这个文件只需下载一次，
	最终结果就是能够加快页面加载的速度。
 
		
//================	<scrpit/> 元素		
	0). 外部的JavaScript文件带有.js扩展名，
	    但这个扩展名不是必须的，
		这样就可以动态的使用JSP PHP或其它语言
		动态生成JavaScript代码也成为了可能，
		/**
			但是服务器通常还是需要看扩展名决定为
			响应使用哪种MIME类型，
			如果不使用js，
			请确保服务器能正确的返回MIME类型
		*/
	0). 多个 <script/> 浏览器
		会按照<script>元素在页面中出现的先后顺序
		对他们依次进行解析		
		
//================	<scrpit/> 元素
	
	1). 包涵在 <script> 元素内部的javascript代码 
		将会从上往下依次解释
		//===
		/**
			在解释器对 <script/> 元素内部的
			所有代码 求值完毕以前，
			页面中的其它内容不会被加载或显示	
		*/
	2). 通过 src 添加的js代码，
		在解析外部JavaScript文件(包括下载该文件)时，
		页面的处理也会暂时停止，
		
	3).浏览器在预到 <body>	标签时才开始呈现内容
		/**
			对于那些有很多Js代码的页面来说，
			这些导致浏览器呈现页面出现延迟
			延迟期间浏览器窗口是一片空白
		*/
		现代web应用程序一般把全部Js引用放
		<body>元素中页面内容的后面。
		这样解析js代码之前，
		页面的内容将完全呈现在浏览器中。
		
//================	<scrpit/> 元素 属性
	
	defer: 延迟加载
		   外部脚本有效，
		   把延迟脚本放在最底部是最佳选择
	
	async：异步脚本，
		   两个异步脚本的内容，不能相互依赖，
