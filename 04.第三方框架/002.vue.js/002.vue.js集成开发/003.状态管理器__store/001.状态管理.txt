//================================================================== vue.js 状态管理

Vue.js 中的 "状态管理器" 主要用于 "管理应用的全局状态"，
使得 "组件之间" 可以方便地 "共享和管理数据"。


最常用的 "状态管理库" 是 Vuex，
它是一个专门为 Vue.js 应用设计的 "状态管理模式" 和 "库"。



//================================================================== 什么是 "状态管理" ？
//================================================================== 什么是 "状态管理" ？

"状态管理" 指的是 "对应用中的状态（数据）" 进行 "集中管理"。

在一个 "复杂的应用" 中，"" 之间可能需要共享数据，
这时候 "简单的父子组件" 通信就显得不够用了。
这就是 "状态管理" 的作用，它提供了一个"集中式的存储"，
用于存储 "所有组件的共享状态"。


//================================================================== 替代方案
//================================================================== 替代方案

虽然 Vuex 是官方推荐的状态管理解决方案，
但对于一些较小的项目或者特定场景，
可以考虑使用简单的事件总线模式、提供者模式（provide/inject），
或者基于 Composition API 的状态管理方法。

通过这些概念和示例，您可以了解到 Vue.js 中状态管理的基本原理和使用方式。
状态管理对于构建复杂的 Vue 应用程序非常重要，它可以帮助您保持应用的可预测性和调试性。
